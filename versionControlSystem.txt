git init: intialize a repository

Purpose: git version control을 처음 사용할때 초기화 해주기 위해서 사용하는 명령어
Details of task: git init 명령어를 실행하면 현재 디렉터리에 git 저장소가 만들어진다.
Results: 새로운 git 저장소가 만들어진다.



git clone: get a copy of an existing Git repository

Purpose: 저장소 받아오기
Details of task: 로컬 저장소를 복제하려면 git clone /로컬/저장소/경로, 원격 서버의 저장소를 복제하려면 git clone htttps://github.com/원격/저장소 명령을 실행한다.
Results: 현재 디렉터리에 원격저장소와 동일한 이름의 폴더가 생성되며 안의 파일들도 그대로 복제되어 진다.



git status: check the status of files (e.g. stagd and unstaged files)

Purpose: 현재의 상태를 확인할 수 있다.
Details of task:  add 또는 commit을 한 후 git status 라는 명령어를 주면 변경된 상태를 확인할 수 있다.
Results: staged 되어 commit될 준비가 완료된 파일들, staged 되지 않은 파일들 그리고 새로 추가된 파일과 수정된 파일의 상태를 보여준다.



git add: track a new file, stage a modified files

Purpose: 변경된 파일 또는 새로운 파일을 추가한다.
Details of task: git add <파일 이름> 
Results: 파일이 staged 상태가 된다. commit을 해줄준비가 되어있다.



git commit: commit the changes of staged files

Purpose: 실제로 변경한 내용을 확정한다. 현재 로컬 디렉터리에 내용 저장.
Details of task: git commit 명령어를 내린다 -m 을 사용하여 추가적으로 메시지를 남딜 수 있다.
Results: 새로 추가되거나 변경되어 add 된 파일들 즉 staged 되어 있는 파일들이 로컬 저장소에 저장된다.

git push: push data to the remote repository

Purpose: 현재 변경된 내용은 로컬저장소에 머물고 있다. 이 변경 내용을 원격 서버로 올린다.
Details of task: git push origin master, 여기서 origin은 원격 저장소의 경로를 remote하여 설정해준 이름이다.
Results: 변경된 내용 그리고 추가된 내용이 원격서버에 올라간다.



git branch: create a new branch
Purpose: 안전하게 격리된 상태에서 무언가를 만들 때 사용한다.
Details of task: 다른 가지를 이용해서 개발을 진행하고, 나중에 개발이 완료되면 master 가지로 돌아와 병합한다.
Results: 두개의 가지가 생긴다.



git checkout: switch to another branch
Purpose: 로컬 변경 내용 되돌리기
Details of task: git checkout -- <파일이름>
Results: 로컬의 변경된 내용을 변경 전 상태로 되돌려 준다. 



git merge: merge the changes in a branch into the current one
Purpose: 로컬 저장소를 원격 저장소에 맞춰 갱신한다.
Details of task: git은 자동으로 변경 내용을 병합하려고 시도한다. branch가 다를 경우에는 merge conflic 충돌이 일어날 수 가 있다. 이때에는 직접 수정해서 병합이 가능하도록 해야한다. 직접 병합을 한 후 git add + git commit 을 수 행하면 merge가 된다.
Results: 서로 다른 두개의 파일이 하나의 동일한 파일로 병합이 된다.
